## Overview

ViewModels in this architecture follow the MVVM pattern and serve as the presentation layer's business logic coordinator. They conform to SwiftUI's `ObservableObject` protocol and manage UI state through `@Published` properties.

## Base pattern

ViewModels are implemented as `final` classes that:

- Inherit from `ObservableObject` for SwiftUI integration
- Use `@Published` properties for reactive state updates
- Depend on use cases from the domain layer
- Are marked with `@MainActor` when they perform async operations
- Contain UI-specific business logic and state transformation

## LoginScreenViewModel

Manages authentication state and user login flow.

### Type signature

```swift
@MainActor
public final class LoginScreenViewModel: ObservableObject {
    public init(userLogin: UserLoginUseCase)
}
```

### Properties

<ParamField path="username" type="String">
  Published property for username input. Bound to the UI text field.
</ParamField>

<ParamField path="password" type="String">
  Published property for password input. Bound to the UI secure field.
</ParamField>

<ParamField path="isLoading" type="Bool">
  Published property indicating async login operation status.
</ParamField>

<ParamField path="error" type="String?">
  Published property containing user-facing error message when login fails.
</ParamField>

### Methods

<ParamField path="login()" type="async -> Void">
  Executes the login use case with current username and password. Updates `isLoading` and `error` states accordingly.
</ParamField>

### Usage example

```swift
import User

@MainActor
let viewModel = LoginScreenViewModel(
    userLogin: userDI.userLoginUseCase
)

// In SwiftUI View
TextField("Username", text: $viewModel.username)
SecureField("Password", text: $viewModel.password)

Button("Login") {
    Task {
        await viewModel.login()
    }
}

if let error = viewModel.error {
    Text(error).foregroundColor(.red)
}
```

### Implementation details

The ViewModel maps domain errors to user-friendly messages:

```swift
private func mapLoginErrorToMessage(_ error: LoginError) -> String {
    switch error {
    case .usernameIsEmpty:
        return "Username is required."
    case .passwordIsEmpty:
        return "Password is required."
    case .invalidCredentials:
        return "Invalid username or password."
    case .unknown:
        return "An unknown error occurred. Please try again later."
    }
}
```

## Architecture principles

### Separation of concerns

ViewModels handle business logic and state management, while navigation is delegated to navigation protocols. This allows ViewModels to be testable without navigation dependencies.

### Dependency injection

ViewModels receive dependencies through constructor injection, typically use cases from the domain layer:

```swift
public init(userLogin: UserLoginUseCase) {
    self.userLogin = userLogin
}
```

### MainActor

ViewModels that perform async operations are marked with `@MainActor` to ensure UI updates happen on the main thread:

```swift
@MainActor
public final class LoginScreenViewModel: ObservableObject {
    func login() async {
        // Safe to update @Published properties
    }
}
```
