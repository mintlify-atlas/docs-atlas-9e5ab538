## Overview

Views in this architecture are SwiftUI structs that conform to the `View` protocol. They follow a strict separation of concerns by delegating business logic to ViewModels and navigation to Navigation protocols.

## Base pattern

Views are implemented as `public struct` types that:

- Conform to SwiftUI's `View` protocol
- Observe ViewModels using `@ObservedObject`
- Receive dependencies through constructor injection
- Handle navigation through injected navigation protocols
- Contain only UI layout and presentation logic

## LoginScreenView

Authentication screen with username/password inputs and login action.

### Type signature

```swift
public struct LoginScreenView: View {
    public init(viewModel: LoginScreenViewModel)
}
```

### Parameters

<ParamField path="viewModel" type="LoginScreenViewModel" required>
  The ViewModel managing authentication state and login logic. Observed for reactive updates.
</ParamField>

### Usage example

```swift
import SwiftUI

let viewModel = LoginScreenViewModel(userLogin: userDI.userLoginUseCase)
let view = LoginScreenView(viewModel: viewModel)
```

### Implementation

```swift
public var body: some View {
    VStack(spacing: 20) {
        TextField("Username", text: $viewModel.username)
            .textInputAutocapitalization(.never)
            .autocorrectionDisabled(true)
        SecureField("Password", text: $viewModel.password)
        if viewModel.isLoading {
            ProgressView()
        } else {
            Button("Login") {
                Task {
                    await viewModel.login()
                }
            }
        }
        if let error = viewModel.error {
            Text(error).foregroundColor(.red)
        }
    }
    .padding()
}
```

## Navigation protocols

## Architecture principles

### Unidirectional data flow

Views observe ViewModels and react to state changes. User actions are sent to ViewModels as method calls:

```swift
Button("Action") {
    viewModel.handleAction()
}
```

### Dependency injection

All dependencies are injected through the initializer:

```swift
public init(viewModel: ViewModel, navigation: Navigation) {
    self.viewModel = viewModel
    self.navigation = navigation
}
```

### Protocol-based navigation

Navigation is abstracted behind protocols, allowing features to navigate without direct dependencies on other features:

```swift
navigation.openCartDetail(id: id)  // Protocol method, implementation in coordinator
```
